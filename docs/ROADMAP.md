# Roadmap

## Phase 0: Project Foundation

- Commit product docs.
- Scaffold app.
- Set up PostgreSQL and Prisma.
- Set up auth skeleton.
- Add CI for lint and tests.
- Create parser fixture structure.

## Phase 1: Prototype

- Family and child setup.
- Calendar source model.
- URL import.
- PDF upload.
- ICS import.
- Basic event extraction.
- Parent review queue.
- Matching engine.
- Basic timeline and recommendations UI.

## Phase 2: Private Beta MVP

- Google login.
- Login with Apple.
- Google Calendar integration.
- Outlook Calendar integration.
- Improved UCLA/Vanderbilt/Saratoga parser support.
- Free-window explanation UI.
- Saved windows.
- Source refresh and change detection.
- Beta feedback capture.

## Phase 3: Hardening

- Better parser confidence scoring.
- Expanded source corpus.
- Error recovery and unsupported source flow.
- OAuth disconnect and data deletion polish.
- Performance tuning for large calendars.
- Accessibility pass.

## Phase 4: Native Mobile Exploration

- Validate whether responsive web covers most use cases.
- Define native mobile requirements.
- Decide between React Native, Swift/Kotlin, or continued web-first approach.

## Key Milestones

| Milestone | Outcome |
|---|---|
| MVP Prototype | A local or hosted prototype can import, review, and match calendars |
| Private Beta | Invited families can use real accounts and real calendars |
| Parser Confidence | Source corpus supports repeatable extraction tests |
| Beta Decision | Decide whether to expand, pivot to calendar assistant, or deepen integrations |
